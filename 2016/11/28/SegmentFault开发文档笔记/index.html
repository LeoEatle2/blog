目录规范

直接置于${root}下的目录称作一级目录。一级目录 必须(MUST) 具有某种职能属性。 除了下面列举的一些常见目录之外，${root}下面也可以放置一些跟项目发布相关的文件，例如build.sh，build.xml，Makefile，Gruntfile等等.

一级目录

  1. src
  2. dep
  3. tool
  4. test
  5. doc
  6. entry
  7. assets

资源目录

模块和加载器规范

建议使用AMD规范
部分参数注意

命名规则

小写＋下划线
目录复数

HTML

soft tab

自动闭合标签不使用/

属性出现顺序

  • class
  • id
  • name
  • data-*
  • src, for, type, href, value , max-length, max, min, pattern
  • placeholder, title, alt
  • aria-*, role
  • required, readonly, disabled
    尽量减少标签数量

CSS, SCSS

soft tab

注释统一使用/* */

类名BEM命名法，id驼峰命名
scss驼峰命名

颜色16进制小写

尽量不要属性简写，margin, padding除外

推荐顺序

Javascript

以下几种情况需要空行：

  • 变量声明后（当变量声明在代码块的最后一行时，则无需空行）
  • 注释前（当注释在代码块的第一行时，则无需空行）
  • 代码块后（在函数调用、数组、对象中则无需空行）
  • 文件最后保留一个空行

注释双斜线后跟一个空格// your comment

变量命名

  • 标准变量采用驼峰式命名（除了对象的属性外，主要是考虑到cgi返回的数据）
  • ‘ID’在变量名中全大写
  • ‘URL’在变量名中全大写
  • ‘Android’在变量名中大写第一个字母
  • ‘iOS’在变量名中小写第一个，大写后两个字母
  • 常量全大写，用下划线连接
  • 构造函数，大写第一个字母
  • jquery对象必须以’$’开头命名

{前一定要有空格

永远不要直接使用undefined进行变量判断；

使用typeof和字符串’undefined’对变量进行判断。
